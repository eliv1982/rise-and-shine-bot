# Остановка бота на сервере и локальный запуск для проверки (Docker)

## 1. Остановить бота на сервере

Подключись к серверу по SSH и в каталоге проекта выполни:

```bash
cd /opt/rise-and-shine   # или путь, где у тебя развёрнут бот
docker compose down
```

- **`docker compose down`** — останавливает контейнер и удаляет его (volume с данными не трогает).
- Чтобы только остановить контейнер без удаления:  
  **`docker compose stop`** — тогда потом можно снова запустить **`docker compose start`**.

Пока бот остановлен на сервере, один и тот же `BOT_TOKEN` можно использовать локально для проверки.

---

## 2. Запуск локально через Docker (для проверки)

На своём компьютере в папке проекта:

```powershell
cd "c:\Users\eliv\Cursor_Projects\Promt engineering\Multimodal\Telegram_bot_Rise_and_Shine"
docker compose up -d --build
```

- **`--build`** — пересобрать образ (нужно после изменений кода).
- **`-d`** — в фоне.

Логи в реальном времени:

```powershell
docker compose logs -f bot
```

Остановка локального запуска:

```powershell
docker compose down
```

---

## 3. Снова запустить бота на сервере

После проверки на локальной машине останови локальный контейнер (`docker compose down`), затем на сервере:

```bash
cd /opt/rise-and-shine
docker compose up -d
```

---

## Кратко

| Где        | Действие              | Команда |
|-----------|------------------------|---------|
| **Сервер** | Остановить бота        | `docker compose down` или `docker compose stop` |
| **Сервер** | Логи                  | `docker compose logs -f bot` |
| **Локально** | Запуск для проверки  | `docker compose up -d --build` |
| **Локально** | Логи                 | `docker compose logs -f bot` |
| **Локально** | Остановить            | `docker compose down` |
| **Сервер** | Запустить снова       | `docker compose up -d` |

**Важно:** не держи бота запущенным и на сервере, и локально с одним и тем же токеном — Telegram не даст двум процессам одновременно обрабатывать одни и те же обновления. Сначала останови на сервере, потом запускай локально; после проверки — наоборот.
